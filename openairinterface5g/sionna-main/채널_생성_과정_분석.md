## 1. 채널 모델 선택 및 초기화

### 1.1 채널 모델 종류

Sionna는 3GPP TR 38.901 표준을 기반으로 한 여러 채널 모델을 제공합니다:

- **UMi (Urban Micro-cell)**: 도시 마이크로셀 환경
- **UMa (Urban Macro-cell)**: 도시 매크로셀 환경
- **RMa (Rural Macro-cell)**: 농촌 매크로셀 환경

**참조 파일:**
- `src/sionna/phy/channel/tr38901/umi.py`
- `src/sionna/phy/channel/tr38901/uma.py`
- `src/sionna/phy/channel/tr38901/rma.py`

### 1.2 채널 모델 초기화

```python
from sionna.phy.channel.tr38901 import UMi, UMa, RMa

# UMi 채널 모델 생성 예시
channel_model = UMi(
    carrier_frequency=3.5e9,      # 반송파 주파수 [Hz]
    o2i_model='low',              # Outdoor-to-Indoor 모델 ('low' 또는 'high')
    ut_array=ut_array,            # UT 안테나 배열
    bs_array=bs_array,            # BS 안테나 배열
    direction='downlink',         # 'downlink' 또는 'uplink'
    enable_pathloss=True,         # 경로 손실 활성화
    enable_shadow_fading=True     # 그림자 페이딩 활성화
)
```

**참조 파일:**

## 3. 토폴로지 생성 및 설정

### 3.1 토폴로지 생성 함수

토폴로지는 네트워크의 기지국(BS)과 사용자 단말(UT)의 위치, 방향, 속도 등을 정의합니다.

**주요 함수:**
- `gen_hexgrid_topology()`: 육각형 그리드 기반 토폴로지 생성
- `gen_single_sector_topology()`: 단일 섹터 토폴로지 생성

**참조 파일:**
- `src/sionna/sys/topology.py` - `gen_hexgrid_topology()` 함수

### 3.2 토폴로지 생성 예시

```python
from sionna.sys import gen_hexgrid_topology

# 육각형 그리드 토폴로지 생성
topology = gen_hexgrid_topology(
    batch_size=1,
    num_rings=1,                  # 링 개수
    num_ut_per_sector=3,         # 섹터당 UT 수
    scenario='umi',              # 시나리오 타입
    min_bs_ut_dist=10.0,         # 최소 BS-UT 거리 [m]
    max_bs_ut_dist=200.0,        # 최대 BS-UT 거리 [m]
    los=True,                    # LoS 강제 설정
    return_grid=True             # 그리드 객체 반환
)

# 반환값:
# - ut_loc: UT 위치 [batch_size, num_ut, 3]
# - bs_loc: BS 위치 [batch_size, num_bs, 3]
# - ut_orientations: UT 방향 [batch_size, num_ut, 3]
# - bs_orientations: BS 방향 [batch_size, num_bs, 3]
# - ut_velocities: UT 속도 [batch_size, num_ut, 3]
# - in_state: 실내/실외 상태 [batch_size, num_ut]
# - los: LoS/NLoS 상태
# - bs_virtual_loc: 가상 BS 위치
```

### 3.3 토폴로지 설정

생성된 토폴로지를 채널 모델에 설정합니다:

```python
# 토폴로지 설정
channel_model.set_topology(
    ut_loc=topology[0],              # UT 위치
    bs_loc=topology[1],              # BS 위치
    ut_orientations=topology[2],     # UT 방향
    bs_orientations=topology[3],     # BS 방향
    ut_velocities=topology[4],       # UT 속도
    in_state=topology[5],           # 실내/실외 상태
    los=topology[6],                 # LoS/NLoS 상태
    bs_virtual_loc=topology[7]      # 가상 BS 위치
)
```

**참조 파일:**
- `src/sionna/phy/channel/tr38901/system_level_channel.py` - `set_topology()` 메서드 (94-156줄)

### 3.4 토폴로지 설정 시 내부 처리

`set_topology()` 메서드가 호출되면:

1. **시나리오 토폴로지 업데이트**: `SystemLevelScenario.set_topology()` 호출

## 4. 채널 생성 과정

채널 생성은 여러 단계를 거쳐 수행됩니다.

### 4.1 채널 생성 호출

```python
# 채널 임펄스 응답 생성
h, delays = channel_model(
    num_time_samples=num_ofdm_symbols,
    sampling_frequency=sampling_freq
)
```

**참조 파일:**
- `src/sionna/phy/channel/tr38901/system_level_channel.py` - `__call__()` 메서드 (172-281줄)

### 4.2 채널 생성 단계별 과정

#### Step 1: LSP (Large Scale Parameters) 생성

LSP는 대규모 채널 특성을 나타내는 파라미터입니다:
- 경로 손실 (Path Loss)
- 그림자 페이딩 (Shadow Fading)
- K-팩터 (K-Factor)
- 지연 확산 (Delay Spread)
- 각도 확산 (Angular Spread)

**참조 파일:**
- `src/sionna/phy/channel/tr38901/lsp.py` - `LSPGenerator` 클래스

```python
# LSP 생성 (내부적으로 수행)
lsp = self._lsp_sampler()  # LSPGenerator 호출
```

#### Step 2: Rays 생성

레이(Rays)는 다중 경로 성분을 나타냅니다:
- 도착 각도 (AoA: Angle of Arrival)
- 출발 각도 (AoD: Angle of Departure)
- 천정 도착 각도 (ZoA: Zenith of Arrival)
- 천정 출발 각도 (ZoD: Zenith of Departure)
- 전력 (Powers)
- 지연 (Delays)
- 교차 편파 비율 (XPR: Cross-Polarization Ratio)

**참조 파일:**
- `src/sionna/phy/channel/tr38901/rays.py` - `RaysGenerator` 클래스

```python
# 레이 생성 (내부적으로 수행)
rays = self._ray_sampler(lsp)  # RaysGenerator 호출
```

#### Step 3: Topology 객체 생성

채널 계수 생성에 필요한 Topology 객체를 생성합니다:

```python
# Topology 객체 생성
topology = Topology(
    velocities=self._scenario.ut_velocities,
    moving_end=moving_end,              # 'tx' 또는 'rx'
    los_aoa=deg_2_rad(self._scenario.los_aoa),
    los_aod=deg_2_rad(self._scenario.los_aod),
    los_zoa=deg_2_rad(self._scenario.los_zoa),
    los_zod=deg_2_rad(self._scenario.los_zod),
    los=self._scenario.los,
    distance_3d=self._scenario.distance_3d,
    tx_orientations=tx_orientations,
    rx_orientations=rx_orientations
)
```

**참조 파일:**
- `src/sionna/phy/channel/tr38901/system_level_channel.py` - 213-222줄

#### Step 4: 채널 임펄스 응답 (CIR) 생성

채널 계수 생성기를 사용하여 채널 임펄스 응답을 생성합니다:

```python
# 채널 임펄스 응답 생성
h, delays = self._cir_sampler(
    num_time_samples,
    sampling_frequency,
    k_factor,      # K-팩터
    rays,          # 레이 정보
    topology,      # 토폴로지
    c_ds           # 클러스터 지연 확산
)
```

**참조 파일:**
- `src/sionna/phy/channel/tr38901/channel_coefficients.py` - `ChannelCoefficientsGenerator` 클래스

#### Step 5: 그림자 페이딩 적용 (Step 12)

3GPP 표준의 Step 12에 따라 그림자 페이딩을 적용합니다:

```python

```

## 5. OFDM 채널 변환

### 5.1 GenerateOFDMChannel 클래스

채널 임펄스 응답을 OFDM 주파수 도메인 채널로 변환합니다.

**참조 파일:**
- `src/sionna/phy/channel/generate_ofdm_channel.py` - `GenerateOFDMChannel` 클래스

### 5.2 OFDM 채널 생성 초기화

```python
from sionna.phy.channel import GenerateOFDMChannel

# OFDM 채널 생성기 초기화
ofdm_channel = GenerateOFDMChannel(
    channel_model=channel_model,      # 채널 모델
    resource_grid=resource_grid,      # 리소스 그리드
    normalize_channel=False           # 채널 정규화 여부
)
```

### 5.3 OFDM 채널 생성 과정

```python
# OFDM 채널 생성
h_freq = ofdm_channel(batch_size)
```

**내부 처리 과정:**

1. **채널 임펄스 응답 샘플링**:
   ```python
   h, tau = self._cir_sampler(
       batch_size,
       self._num_ofdm_symbols,
       self._sampling_frequency
   )
   ```

2. **서브캐리어 주파수 계산**:
   ```python
   self._frequencies = subcarrier_frequencies(
       self._num_subcarriers,
       self._subcarrier_spacing,
       self.precision
   )
   ```

3. **CIR을 OFDM 채널로 변환**:
   ```python
   h_freq = cir_to_ofdm_channel(
       self._frequencies,  # 서브캐리어 주파수
       h,                  # 채널 임펄스 응답
       tau,                # 경로 지연
       self._normalize_channel
   )
   ```

**참조 파일:**
- `src/sionna/phy/channel/utils.py` - `cir_to_ofdm_channel()` 함수

### 5.4 OFDM 채널 변환 수식

채널 주파수 응답은 다음 수식으로 계산됩니다:

$$\widehat{h}_{s, n} = \sum_{m=0}^{M-1} a_{m}(s) e^{-j2\pi n \Delta_f \tau_{m}}$$

여기서:
- $\widehat{h}_{s, n}$: $s$번째 OFDM 심볼, $n$번째 서브캐리어의 채널 주파수 응답
- $a_{m}(s)$: $m$번째 경로의 복소 진폭
- $\tau_{m}$: $m$번째 경로의 지연
- $\Delta_f$: 서브캐리어 간격

**참조 파일:**
- `src/sionna/phy/channel/generate_ofdm_channel.py` - 21-22줄

### 5.5 OFDM 채널 출력

- **h_freq**: 채널 주파수 응답 [batch_size, num_rx, num_rx_ant, num_tx, num_tx_ant, num_ofdm_symbols, num_subcarriers]

```python
# OFDM 채널 적용 블록 초기화

```

### 6.3 채널 적용 과정

```python
# 채널 적용
y = apply_channel(x, h_freq, no)
```

**내부 처리:**

1. **입력 신호 차원 확장**:
   ```python
   x = expand_to_rank(x, h_freq.shape.rank, axis=1)
   ```

2. **채널 응답 적용**:
   ```python
   y = tf.reduce_sum(tf.reduce_sum(h_freq * x, axis=4), axis=3)
   ```

3. **AWGN 추가 (선택적)**:
   ```python
   if no is not None:
       y = self._awgn(y, no)
   ```

**참조 파일:**
- `src/sionna/phy/channel/apply_ofdm_channel.py` - `call()` 메서드 (70-80줄)

### 6.4 채널 적용 수식

각 OFDM 심볼 $s$와 서브캐리어 $n$에 대해:

$$y_{s,n} = \widehat{h}_{s, n} x_{s,n} + w_{s,n}$$

여기서:
- $y_{s,n}$: 채널 출력
- $\widehat{h}_{s, n}$: 채널 주파수 응답
- $x_{s,n}$: 채널 입력
- $w_{s,n}$: 가산성 백색 가우시안 잡음 (AWGN)

**참조 파일:**

## 8. 전체 흐름도

```
1. 채널 모델 선택
   └── UMi/UMa/RMa 선택
   
2. 안테나 배열 설정
   ├── UT 안테나 배열 (Antenna/PanelArray)
   └── BS 안테나 배열 (Antenna/PanelArray)
   
3. 채널 모델 초기화
   └── SystemLevelChannel.__init__()
   
4. 토폴로지 생성
   └── gen_hexgrid_topology() 또는 gen_single_sector_topology()
   
5. 토폴로지 설정
   └── channel_model.set_topology()
       ├── SystemLevelScenario.set_topology()
       ├── LSPGenerator.topology_updated_callback()
       ├── RaysGenerator.topology_updated_callback()
       └── LSP 샘플링 (필요시)
   
6. 채널 생성
   └── channel_model(num_time_samples, sampling_frequency)
       ├── Step 1: LSP 생성 (LSPGenerator)
       ├── Step 2: Rays 생성 (RaysGenerator)
       ├── Step 3: Topology 객체 생성
       ├── Step 4: CIR 생성 (ChannelCoefficientsGenerator)
       └── Step 5: 그림자 페이딩 적용 (Step 12)
   
7. OFDM 채널 변환
   └── GenerateOFDMChannel.__call__()
       ├── CIR 샘플링
       ├── 서브캐리어 주파수 계산
       └── cir_to_ofdm_channel() 변환
   
8. 채널 적용
   └── ApplyOFDMChannel.__call__(x, h_freq, no)
       ├── 채널 응답 적용
       └── AWGN 추가 (선택적)
```

## 9. 사용 예시

### 9.1 완전한 예시 코드

```python
import tensorflow as tf
from sionna.phy.channel.tr38901 import UMi, Antenna, PanelArray
from sionna.phy.channel import GenerateOFDMChannel, ApplyOFDMChannel
from sionna.sys import gen_hexgrid_topology
from sionna.phy.ofdm import ResourceGrid

# 1. 리소스 그리드 설정
rg = ResourceGrid(
    num_ofdm_symbols=14,
    fft_size=1024,
    subcarrier_spacing=15e3
)

# 2. 안테나 배열 설정
ut_array = Antenna(
    polarization="single",
    polarization_type="V",
    antenna_pattern="omni",
    carrier_frequency=3.5e9
)

bs_array = PanelArray(
    num_rows_per_panel=4,
    num_cols_per_panel=4,
    polarization='dual',
    polarization_type='VH',
    antenna_pattern='38.901',
    carrier_frequency=3.5e9
)

# 3. 채널 모델 생성
channel_model = UMi(
    carrier_frequency=3.5e9,
    o2i_model='low',
    ut_array=ut_array,
    bs_array=bs_array,
    direction='downlink',
    enable_pathloss=True,
    enable_shadow_fading=True
)

# 4. 토폴로지 생성
topology = gen_hexgrid_topology(
    batch_size=1,
    num_rings=1,
    num_ut_per_sector=3,
    scenario='umi',
    min_bs_ut_dist=10.0,
    max_bs_ut_dist=200.0,
    los=True,
    return_grid=True
)

# 5. 토폴로지 설정
channel_model.set_topology(*topology)

# 6. OFDM 채널 생성
ofdm_channel_gen = GenerateOFDMChannel(
    channel_model=channel_model,
    resource_grid=rg,
    normalize_channel=False
)

h_freq = ofdm_channel_gen(batch_size=1)

# 7. 채널 적용
apply_channel = ApplyOFDMChannel()

# 입력 신호 생성 (예시)
x = tf.complex(
    tf.random.normal([1, 1, 1, 14, 1024]),
    tf.random.normal([1, 1, 1, 14, 1024])
)

# 채널 적용
y = apply_channel(x, h_freq, no=0.1)
```

## 10. 주요 수식 및 파라미터

### 10.1 채널 주파수 응답

$$\widehat{h}_{s, n} = \sum_{m=0}^{M-1} a_{m}(s) e^{-j2\pi n \Delta_f \tau_{m}}$$

### 10.2 채널 출력

$$y_{s,n} = \widehat{h}_{s, n} x_{s,n} + w_{s,n}$$

### 10.3 주요 파라미터

| 파라미터 | 설명 | 단위 |
|---------|------|------|
| `carrier_frequency` | 반송파 주파수 | Hz |
| `num_time_samples` | 시간 샘플 수 | - |
| `sampling_frequency` | 샘플링 주파수 | Hz |
| `num_ofdm_symbols` | OFDM 심볼 수 | - |
| `fft_size` | FFT 크기 (서브캐리어 수) | - |
| `subcarrier_spacing` | 서브캐리어 간격 | Hz |
| `num_rx` | 수신기 수 | - |
| `num_tx` | 송신기 수 | - |
| `num_rx_ant` | 수신 안테나 수 | - |
| `num_tx_ant` | 송신 안테나 수 | - |

## 12. 요약

Sionna의 채널 생성 과정은 다음과 같이 요약할 수 있습니다:

1. **초기화 단계**: 채널 모델 선택 및 안테나 배열 설정
2. **토폴로지 설정**: 네트워크 토폴로지 생성 및 채널 모델에 설정
3. **채널 생성**: LSP → Rays → CIR → 그림자 페이딩 적용
4. **OFDM 변환**: CIR을 주파수 도메인 OFDM 채널로 변환
5. **채널 적용**: 생성된 채널을 신호에 적용

전체 과정은 3GPP TR 38.901 표준을 엄격히 따르며, TensorFlow를 기반으로 GPU 가속을 지원합니다.
